HTML Lesson

HTML is an acronym for Hypertext Markup Language. It is used to build the structure of web pages. The content that you read now is a product of HTML. The simple analogy I give about HTML is that it is like the skeleton of web pages made up of only texts, and multi-media files with no design. However, all mobile/web applications have HTML elements(which you will learn in subsequent lesson).

This lesson focuses on HTML5 which is an upgrade on HTML4. The HTML Document Structure can be seen in the image below.

  1. !DOCTYPE: a HTML5 declaration which means that the document type is HTML5 and not just HTML
  2. html: a general declaration of html
  3. head: links to CSS files, meta data and sometimes javascript files are put there
  4. title: the title of a HTML document is written here which appears on a browser's tab
  5. body: content of a web page is written here

All of the subjects in the list of the previous lesson are called HTML tags. These tags consist of an opening tag/start tag and closing tag/end tag.

An HTML element consist of the following:

  • Opening Tag/Start Tag
  • Closing Tag/End Tag
  • Content. It is only the content of an HTML element that is displayed in the browser.

It is a general practice for developers to write comments when building applications as individuals or group. The essence of the comment is to enable developers or their colleagues to be able to read their codes easily without having to waste time trying to figure out what is where. Also, comments are not displayed in the browser and only the developer sees them. See example in the figure below.

Some HTML tags have only an opening tag and no closing tag; and they called the self closing/void tags.

When developing applications, sometimes a developer is required to write a heading in a section of a web page. So, the heading tags come handy. These heading tags include h1, h2, h3, h4, h5 and h6. The h1 is the biggest heading while h6 is the smallest heading.

The paragrpah tag usually houses lengthy text content in a web page. So, when you have an article or any other lengthy content to add to your web page, use the paragraph tag.

Although it is a general practice to format text using CSS (Cascading Style Sheet), sometimes developers may want a quick solution, hence they use some HTML tags to qiuckly format text. See the figure below for some text formating tags

The image tag is used to add image to a web page. It is self closing tag. To add an image, we add the src (source) attribute to the image. The src attribute defines the path to our image.

HTML attribute provides additional information about an HTML element which is in key/value pair. For example, the src="path-to-iamge-file" is an example of an attribute in the image tag. In this case, the src is the key while "path-to-iamge-file" is the value. In some cases, an html tag could have multiple attributes which are written in the opening tag of an HTML element.

To make a list of items, we use the list tags which could be ordered (ol tag) or unordered (ul tag)list. The ordered list is usually serialized in numbers of letters while the unordered list is not.

The HTML table is used to present tabular data. Although in the past, it was also used for creating navigation menu for applications; it is now considered an archaic thing to do. The Table is wrapped within table tag.

In mobile/web development, the html form is used to collect users information. When you create a facebook account, apply for admission/job register online for an examination, you have simply filled in an HTML form. Within the form, we have input tags with type attribute and values of text, email, password, number, radio, checkbox etc. The form could also have label tags to label the input types. The last tag is always the button/submit which users click to GET/POST the informations entered in the form.

The anchor/a tag is used to link to one page to another either within the same (internal link) application or to another (external link). The link is defined in the href attribute. See figure below.

To group similar HTML elements, the div or span tags are mostly used. The div and span are like contaiers for holding similar group of elements

In the previous lesson, we saw that the within the form tag, other tags (input, label, select) exist. So, those tags are the direct children of the form tag; in other words, they are nested elements of the form. The li as seen in the lesson on HTML List, are also nested elements of the ol and ul tags respectively.

It is important to remember that it is the content of the HTML elements that is output in the browser and not the tags themselves (which are the codes responsible for the content displayed in the browser).
